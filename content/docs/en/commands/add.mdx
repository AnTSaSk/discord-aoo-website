The `/add` command reports a new Black Zone objective to your server's tracking list. All server members can use this command.

## Syntax

```
/add type:<type> rarity:<rarity> map:<map> time:<HH:MM>
```

## Parameters

All four parameters are **required**.

| Parameter | Type | Description |
|-----------|------|-------------|
| `type` | Autocomplete | The objective type — one of 6 values (see [objective types](/docs/guides/objective-types/)) |
| `rarity` | Autocomplete | Tier for nodes (`4.4`–`8.4`) or color for cores/vortexes (`Green`, `Blue`, `Purple`, `Gold`) |
| `map` | Autocomplete | Black Zone map name — 276 maps supported (see [maps list](/docs/guides/black-zone-maps/)) |
| `time` | Text | Time remaining until the objective spawns, format `HH:MM` (24-hour). Example: `02:30` (2 hours 30 minutes from now) |

### Type Values

| Type | Category |
|------|----------|
| `Fiber node` | Node |
| `Hide node` | Node |
| `Ore node` | Node |
| `Wood node` | Node |
| `Core` | Core |
| `Vortex` | Vortex |

### Rarity Values

**For nodes:** `4.4`, `5.4`, `6.4`, `7.4`, `8.4`

**For cores and vortexes:** `Green`, `Blue`, `Purple`, `Gold`

## Autocomplete

The `type`, `rarity`, and `map` parameters all support Discord's autocomplete feature. Start typing and matching options will appear. For maps, the search matches anywhere in the name — beginning, middle, or end.

Additionally, the `type` and `rarity` fields are context-aware: selecting a node rarity (4.4–8.4) filters the type list to nodes only, and selecting a core or vortex type filters rarities to colors only.

## Behavior

### Duplicate Prevention

<Callout variant="info">Albion Online Objectives prevents duplicates automatically. If the same objective type, rarity, and map is reported within 8 minutes, the bot rejects it.</Callout>

If an objective with the same type, rarity, and map already exists within an **8-minute window** (±4 minutes of the reported time), the bot rejects the new report and notifies you. This prevents accidental duplicates when multiple guild members spot the same objective.

### Maintenance Detection

<Callout variant="warning">When the calculated spawn time crosses the daily 10:00 UTC maintenance window, the bot flags the objective with a warning.</Callout>

Albion Online's daily server maintenance occurs at **10:00 UTC**. If the calculated spawn time crosses the **10:00 UTC** daily maintenance window, the bot automatically flags it. This tells your guildmates that the objective may not survive the server reset.

### Cooldown

This command has a **2-second cooldown** per user.

### Time Validation

The `time` parameter must match the `HH:MM` 24-hour format (e.g., `01:30`, `04:00`, `12:45`). Invalid formats are rejected.

## After Reporting

Once added, the objective appears in the [`/list`](/docs/commands/list/) output. To remove an incorrect entry, use [`/remove`](/docs/commands/remove/).
